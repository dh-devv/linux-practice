# Ubuntu Linux 학습 종합 일지 (Learning Log)

## 1. 개요
본 일지는 우분투 리눅스(Ubuntu Linux) 환경에서 진행한 기본 명령어 학습부터 파일 및 디렉터리 제어, 네트워크(DNS) 구조, 텍스트 파이프라인, 셸 스크립트 기초, 그리고 디스크 마운트와 프로세스 관리에 이르는 전체 실습 과정을 종합 정리한 문서입니다.

---

## 2. 일차별 상세 학습 기록

### Day 01: 기본 명령어 및 시스템 제어
* **주요 학습 내용**:
  * 파일/디렉터리 탐색 명령어 (`ls`, `pwd`, `cd`, `cd ..`)
  * 시스템 종료 및 재부팅 명령어 (`init 0/6`, `shutdown`, `reboot`, `poweroff`)
* **배운 점**:
  * 리눅스 터미널의 기본 이동 구조와 현재 위치 확인 방식(`pwd`)을 습득함.
  * 시스템을 종료할 때 단순히 전원을 끄는 것이 아니라 런레벨(Runlevel) 변경(`init`)이나 안전한 신호 전달(`shutdown`)을 통해 프로세스를 정돈하고 종료해야 파일 시스템 손상을 방지할 수 있음을 이해함.
* **어려웠던 점 및 해결**:
  * `init 0`과 `shutdown -h`의 차이에 대해 의문이 생김. `init`은 시스템 초기화 프로세스에 런레벨 변경을 직접 지시하는 방식이고, `shutdown`은 접속 유저에게 알림 메세지(`-k`)를 보내거나 예약을 거는 등 사용자 친화적 제어가 가능하다는 차이를 확인 및 정리함.

---

### Day 02: 폴더 및 파일 다루기
* **주요 학습 내용**:
  * 디렉터리 및 파일 생성 (`mkdir`, `touch`)
  * 화면 정리 및 파일 삭제 (`clear`, `rm`)
  * 텍스트 에디터 및 내용 확인 (`nano`, `cat`)
* **배운 점**:
  * CUI 환경에서 파일을 생성하고 편집기(`nano`)를 통해 내용을 작성한 뒤 `cat`으로 확인하는 일련의 파일 제어 흐름을 익힘.
* **어려웠던 점 및 해결**:
  * 상대 경로와 절대 경로 지정 시 위치를 헷갈려 에러가 발생함. 디렉터리 작업 전 항상 `pwd`로 위치를 확인하고 `cd` 이동 시 경로 구분을 명확히 하는 습관을 들임. 또한 `nano` 에디터 작성 후 터미널 복귀 시 저장 키(`Ctrl+O`, `Ctrl+X`) 동작 순서를 확실히 숙지함.

---

### Day 03: 네트워크 기초 (DNS와 네임 서버)
* **주요 학습 내용**:
  * DNS(Domain Name System)의 개념과 역할
  * Hosts 파일의 우위 참조 원리 및 네임 서버 구조
  * DNS 캐시 및 계층적(Root, TLD, Authoritative) IP 조회 과정
* **배운 점**:
  * 사람이 기억하기 쉬운 도메인을 숫자로 된 IP 주소로 변환하는 트리 구조의 DNS 매커니즘을 이해함.
  * 도메인 접속 시 외부 DNS 서버로 요청이 나가기 전, 로컬의 `/etc/hosts` 파일과 DNS 캐시를 먼저 조회한다는 우선순위를 파악함.
* **어려웠던 점 및 해결**:
  * 단순 1대1 변환이 아니라 루트(`.`) -> `.com` -> `naver.com`으로 이어지는 계층 구조라는 점이 처음에는 복잡하게 느껴짐. 단계별 질의 과정을 순서대로 정리해보면서 실제 웹 브라우저 접속 시 발생하는 네트워크 비하인드 동작을 명확히 이해함.

---

### Day 04: 텍스트 파일 출력 및 파이프라인 활용
* **주요 학습 내용**:
  * 텍스트 출력 명령어 (`cat`, `head`, `tail`)
  * 대용량 파일 페이징 탐색 (`more`, `less`)
  * 리다이렉션(`>`) 및 파이프(`|`) 연산자
* **배운 점**:
  * 파일 전반부/후반부만 확인할 때 `head`와 `tail`을 사용하는 효율성을 확인함 (특히 로그 파일 분석 시 `tail` 활용도가 높음).
  * 앞 명령어의 실행 결과를 뒤 명령어의 입력으로 전달하는 파이프(`|`)와 파일로 저장하는 리다이렉션(`>`)의 강력함을 체감함.
* **어려웠던 점 및 해결**:
  * `more`와 `less`의 기능적 차이가 헷갈렸음. `more`는 스크롤 다운 위주의 단순 분할 출력인 반면, `less`는 방향키를 이용한 상하 이동 및 버퍼 탐색이 자유롭다는 점을 실습을 통해 비교 정리함.

---

### Day 05: 셸 스크립트 기초 및 실행 권한
* **주요 학습 내용**:
  * 셸 스크립트 작성 (`#!/bin/sh`, `echo`, `exit 0`)
  * 환경 변수 활용 (`$USER`, `$HOME`)
  * 스크립트 실행 방식 (`sh script.sh` vs `chmod +x` 후 `./script.sh`)
* **배운 점**:
  * 반복되는 명령어 작업을 자동화할 수 있는 셸 스크립트의 기본 구조(Shebang 설정, 변수 호출, 종료 코드 반환)를 학습함.
* **어려웠던 점 및 해결**:
  * `./name.sh`로 직접 실행 시 `Permission denied` 에러가 발생함. 리눅스 파일 시스템에서는 기본 생성된 파일에 실행(`x`) 권한이 부여되지 않음을 파악하고, `chmod +x` 명령어로 권한을 부여한 뒤 실행하는 원리를 체득함.

---

### Day 06: 디스크 마운트, 사용자 관리 및 프로세스 제어
* **주요 학습 내용**:
  * 장치 마운트 개념 (`mount`, `umount`, `/dev/` 장치 파일)
  * 디스크 용량 확인 (`df -h`) 및 파일 검색 (`grep`, `diff`, 정규표현식)
  * 사용자 계정 관리 (`useradd`, `usermod`, `passwd`, `userdel`, `users`) 및 단축 명령어 (`alias`, `which`)
  * 프로세스 관리 (`ps aux`, `pstree`, `jobs`, `bg`/`fg`, `kill -9`, `nohup`)
* **배운 점**:
  * 윈도우의 드라이브 문자(C:, D:) 방식과 달리, 리눅스는 최상위 루트(`/`) 디렉터리 트리의 특정 빈 폴더에 실물 장치를 연결하여 쓰는 마운트(Mount) 방식을 채택한다는 점을 깊이 이해함.
  * 현재 실행 중인 프로그램(프로세스)의 PID를 조회하고, 필요에 따라 시그널(`kill -9`)을 보내 제어하거나 백그라운드(`&`, `bg`, `fg`) 작업으로 전환하는 시스템 관리자 수준의 조작법을 익힘.
* **어려웠던 점 및 해결**:
  * '모든 것은 파일이다(Everything is a file)'라는 리눅스 철학상 장치(`dev/sr0` 등)도 파일로 존재한다는 점과 마운트 폴더의 관계가 혼동되었으나, 실제 `df -h`로 연결 상태를 확인하고 `mount/umount`를 직접 수행하면서 개념을 정립함.
  * `ps` 명령어의 수많은 옵션 중 `aux`의 조합 의미(`a`: 모든 유저, `u`: 상세 정보, `x`: 터미널 미할당 프로세스 포함)를 하나씩 파헤쳐 이해함.

---

## 3. 종합 회고 및 학습 성과

1. **CUI 환경에 대한 적응력 향상**
   * GUI 클릭 방식에서 벗어나 명령어와 키보드 위주의 터미널 작업에 익숙해졌으며, 디렉터리 이동 및 파일 편집을 신속하게 수행할 수 있게 됨.
2. **시스템 및 네트워크 구조적 이해**
   * 단순 명령어 암기를 넘어 리눅스의 파일 시스템 구조(루트 디렉터리 체계, 마운트 원리), DNS 응답 흐름, 프로세스 상태 주기 등 원리 중심의 학습을 이룸.
3. **문제 해결(Troubleshooting) 능력 개발**
   * 권한 문제(`Permission denied`), 경로 에러, 프로세스 종료 등의 상황에서 원인을 분석하고 `chmod`, `ps`, `kill` 등을 적절히 조합하여 해결하는 능력을 기름.

## 4. 향후 학습 계획
* **사용자 및 그룹 권한 심화**: `chmod` (8진수 표기법), `chown`, `chgrp` 및 Special Permission(SetUID, SetGID, Sticky Bit) 학습
* **네트워크 서비스 구축**: DNS(Bind9), Web(Apache/Nginx) 서버 실제 구성 실습
* **셸 스크립트 응용**: 조건문(`if`), 반복문(`for`/`while`) 및 조건 비교를 활용한 시스템 자동 점검 스크립트 작성
