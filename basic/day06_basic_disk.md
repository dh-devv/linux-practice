# Day 06 - 리눅스 프로세스(Process) 관리

## 1. 오늘 배운 주요 개념 및 명령어

### 프로세스 (Process) 개요
* **프로세스**: 현재 메모리에 올려져 실행 중인 프로그램.
* **포그라운드 (Foreground)**: 눈에 보이는 작업 공간. 터미널을 점유하고 있어 작업이 끝날 때까지 다른 명령을 입력할 수 없음.
* **백그라운드 (Background)**: 눈에 보이지 않게 뒤에서 실행되는 작업. 다른 작업을 동시에 진행할 수 있음 (명령어 뒤에 `&` 붙여서 실행).
* `fork()`: 기존 프로세스를 복제하여 새로운 프로세스를 생성하는 시스템 콜 (메모리 할당 이루어짐).
* **데몬 프로세스 (Daemon)**: 부팅 시 백그라운드로 실행되는 서버 프로세스 (DB, DNS 등).
  * **standalone 데몬**: 독립적으로 계속 실행되는 데몬.
  * **inetd 데몬 (슈퍼 데몬)**: 요청이 올 때만 프로세스를 연결해 주며 통합 관리하는 데몬.
* **시그널 (Signal)**: 프로세스를 제어 및 관리하기 위해 전달하는 신호.

### 프로세스 조회 및 제어 명령어
* `ps`: 현재 실행 중인 프로세스 목록 출력.
  * `a`: 현재 실행 중인 모든 사용자 프로세스 출력
  * `e`: 모든 프로세스 정보 출력
  * `u`: 프로세스 소유자(사용자) 및 시작 시간 등 상세 정보 출력
  * `x`: 접속한 터미널에 할당되지 않은 모든 프로세스까지 출력
  * `l`: 자세한 형식으로 출력
  * `ps aux`: 가장 흔하게 사용하는 조합 예시
* `pstree`: 프로세스들의 부모-자식 계층 관계를 트리 구조로 시각화.
* `jobs`: 현재 셸이 관리 중인 백그라운드 작업 목록 확인.
* `bg` / `fg`: 실행 중인 프로세스의 상태 전환 (`bg`: 백그라운드로 전환, `fg`: 포그라운드로 가져오기).
* `kill`: 프로세스에 시그널을 보내 종료하는 명령어.
  * `-1`: 프로세스 재시작 (상태 재설정)
  * `-9`: 프로세스 무조건 강제 종료 (예: `kill -9 [PID]`)
* `killall`: 동일한 이름의 여러 프로세스를 한 번에 일괄 종료.
* `nice`: 프로세스 실행 시 우선순위(nice 값)를 지정.
* `renice`: 이미 실행 중인 프로세스의 우선순위 변경.
* `nohup`: 터미널 세션이 닫혀도 프로세스가 종료되지 않고 계속 백그라운드에서 실행되도록 유지.

## 2. 실습 기록 (Practice Log)

### 실습 1) 프로세스 상태 확인 및 트리 구조 출력
> `ps aux`로 전체 프로세스 상위 정보를 확인하고, `pstree`로 계층 구조를 출력하는 실습

```bash
# 1. 시스템 내 모든 프로세스의 상세 정보 확인 (상위 5개만 출력)
$ ps aux | head -n 5
USER         PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root           1  0.0  0.4 170816  9520 ?        Ss   09:00   0:01 /usr/lib/systemd/systemd
root           2  0.0  0.0      0     0 ?        S    09:00   0:00 [kthreadd]

# 2. 프로세스 계층 구조를 트리 형태로 보기
$ pstree
systemd─┬─NetworkManager
        ├─sshd───sshd───bash───pstree
        └─tuned
```

###  실습 2) 포그라운드/백그라운드 전환 및 작업 제어 (`jobs`, `bg`, `fg`)
> 프로세스를 백그라운드로 실행하고 작업 상태를 일시 중지 및 전환하는 실습

```bash
# 1. sleep 명령어를 백그라운드(&)에서 실행
$ sleep 500 &
[1] 3421

# 2. 현재 셸이 관리 중인 백그라운드 작업 확인
$ jobs
[1]+  Running                 sleep 500 &

# 3. 백그라운드 작업([1]번)을 포그라운드로 전환 후 Ctrl+Z로 일시 정지
$ fg %1
sleep 500
^Z
[1]+  Stopped                 sleep 500

# 4. 일시 정지된(Stopped) 작업을 다시 백그라운드에서 실행
$ bg %1
[1]+ sleep 500 &
```

### 실습 3) 프로세스 강제 종료 (`kill -9`)
> 실행 중인 프로세스의 PID를 확인하고 시그널을 보내 강제 종료하는 실습

```bash
# 1. sleep 프로세스의 PID 확인 (PID: 3421)
$ ps aux | grep sleep
user1     3421  0.0  0.0   8300   816 pts/0    S    10:15   0:00 sleep 500

# 2. kill -9 옵션으로 3421번 프로세스 강제 종료
$ kill -9 3421
[1]+  Killed                  sleep 500

# 3. 정상적으로 종료되었는지 확인
$ jobs
# (출력 없음 - 프로세스가 정상적으로 정리됨)
```
