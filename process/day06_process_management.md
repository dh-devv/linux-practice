# Linux 프로세스(Process) 관리

## 1. 개념 및 명령어 정리
* **프로세스**: 실행 중인 프로그램입니다.
* **백그라운드**: 눈에 보이지 않게 실행되는 프로세스로, 다른 작업을 동시에 진행할 수 있습니다.
* **포그라운드**: 눈에 보이는 프로세스로, 작업 중에는 다른 작업을 할 수 없습니다[cite: 4].
* `fork()`: 기존 프로세스를 복제하며 메모리 할당이 이루어집니다[cite: 4].
* **데몬 프로세스**: 부팅되면서 실행되는 프로세스로 DB, DNS 등 서버 프로그램이 해당됩니다[cite: 4].
  * **standalone 데몬**: 독립적으로 실행할 수 있는 데몬 프로세스입니다[cite: 4].
  * **inetd 데몬 (슈퍼 데몬)**: 프로세스를 통합 관리합니다[cite: 4].
* **시그널**: 시그널을 이용해서 프로세스를 관리 및 제어합니다[cite: 4].
* `ps`: 현재 실행 중인 프로세스 목록을 보여줍니다[cite: 4].
  * `a`: 현재 실행 중인 모든 프로세스 출력[cite: 4]
  * `e`: 모든 프로세스 정보[cite: 4]
  * `u`: 사용자 이름과 프로세스 시작 시간 출력[cite: 4]
  * `x`: 사용되고 있는 모든 프로세스 출력 (접속한 터미널 외 모두 포함)[cite: 4]
  * `l`: 자세한 정보 출력[cite: 4]
  * `ps aux`: 많이 사용하는 조합 예시[cite: 4]
* `pstree`: 실행 중인 프로세스들을 트리 구조로 보여줍니다[cite: 4].
* `jobs`: 현재 셸이 관리하는 프로세스의 정보를 보여줍니다[cite: 4].
* `bg` / `fg`: 실행 중인 프로세스의 상태(백그라운드/포그라운드)를 전환합니다[cite: 4].
* `kill`: 프로세스를 종료하는 명령어입니다[cite: 4].
  * `-1`: 재시작[cite: 4]
  * `-9`: 강제 종료 (예: `kill -9 [PID]`)[cite: 4]
* `killall`: 여러 프로세스를 한 번에 종료할 때 사용합니다[cite: 4].
* `nice`: 프로세스의 우선순위를 변경하여 실행합니다[cite: 4].
* `rnice` (`renice`): 이미 실행 중인 프로세스의 nice 값을 변경합니다[cite: 4].
* `nohup`: 터미널이 종료되어도 프로세스가 끊기지 않고 계속 실행되도록 합니다[cite: 4].

## 2. 실습 기록 (Practice Log)

### 실습 1) 프로세스 상태 확인 및 트리 구조 출력
전체 프로세스 상태와 계층 구조를 조회하는 실습을 진행함[cite: 4].

```bash
# 1. 시스템 내 모든 프로세스의 상세 정보 확인
$ ps aux | head -n 5
USER         PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root           1  0.0  0.4 170816  9520 ?        Ss   09:00   0:01 /usr/lib/systemd/systemd
root           2  0.0  0.0      0     0 ?        S    09:00   0:00 [kthreadd]

# 2. 프로세스 트리 구조 확인
$ pstree
systemd─┬─NetworkManager
        ├─sshd───sshd───bash───pstree
        └─tuned
```


### 실습 2) 포그라운드/백그라운드 전환 및 작업 제어 (jobs, bg, fg)
프로세스를 백그라운드로 실행하고 작업 상태를 전환하는 실습을 진행함

```bash
# 1. sleep 명령어를 백그라운드(&)에서 실행
$ sleep 500 &
[1] 3421

# 2. 관리 중인 백그라운드 작업 확인
$ jobs
[1]+  Running                 sleep 500 &

# 3. 백그라운드 작업을 포그라운드로 전환
$ fg %1
sleep 500
^Z
[1]+  Stopped                 sleep 500

# 4. 일시 정지된 작업을 다시 백그라운드에서 실행
$ bg %1
[1]+ sleep 500 &
```

### 실습 3) 프로세스 강제 종료 (kill -9)
실행 중인 프로세스의 PID를 확인하고 시그널을 보내 강제 종료하는 실습을 진행함

```bash
# 1. sleep 프로세스의 PID 확인
$ ps aux | grep sleep
user1     3421  0.0  0.0   8300   816 pts/0    S    10:15   0:00 sleep 500

# 2. kill -9 옵션으로 프로세스 강제 종료
$ kill -9 3421
[1]+  Killed                  sleep 500

# 3. 종료 여부 재확인
$ jobs
# (출력 없음 - 정상 종료됨)
```
